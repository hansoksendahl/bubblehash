BubbleHash

A distributed-hash-table implemeted using IndexedDB and WebRTC.

Try It! Learn More
BubbleHashconstructorvar bubblehash = new BubbleHash([id], [options]);

A BubbleHash instance creates a Chord distributed hash table implemented using peer.js.

[id]string

The id of some other peer.

[options]object
[log]boolean

Sets whether BubbleHash should log messages sent between clients. Note: setting options.log to true can produce an alarming amount of console output.

[peer]object

Options for the underlying peer.js connections.

portnumber

Server port. Defaults to 80.

keystring

API key for the cloud PeerServer. This is not used for servers other than 0.peerjs.com.PeerServer cloud runs on port 9000. Please ensure it is not blocked or consider running your own PeerServer instead.

pathstring

The path where your self-hosted PeerServer is running. Defaults to '/'.

hoststring

Server host. Defaults to 0.peerjs.com. Also accepts '/' to signify relative hostname.

secureboolean

true if you're using SSL.Note that our cloud-hosted server and assets may not support SSL.

configobject

Configuration hash passed to RTCPeerConnection. This hash contains any custom ICE/TURN server configuration. Defaults to { 'iceServers': [{ 'url': 'stun:stun.l.google.com:19302' }] }

debugnumber

Prints log messages depending on the debug level passed in. Defaults to 0.

0

Prints no logs.

1

Prints only errors.

2

Prints errors and warnings.

3

Prints all logs.

[interval]object
[stabilize]number

The interval (in seconds) for the stabilize process.

[fixFingers]number

The interval (in seconds) for the fixFingers process.

[checkPredecessor]number

The interval (in seconds) for the checkPredecesssors process.

[fixSuccessorList]number

The interval (in seconds) for the fixSuccessorList process.

[fixSuccessor]number

The interval (in seconds) for the fixSuccessors process.

bubblehash.joinmethodvar dataConnection = bubblehash.join(id);

Connects to the remote peer at id. For more information on the events and methods attached to the dataConnection object please see the peer.js documentation.

bubblehash.onmethodbubblehash.on(event, callback);

Set listeners for BubbleHash events.

messageeventbubblehash.on("message", function () {/*...*/});

Emitted when BubbleHash receives a message intended for the client.

emptyeventbubblehash.on("empty", function () {/*...*/});

Emitted when BubbleHash has no successor or predecessor defined and no entries in its finger table.

bubblehash.destroymethodbubblehash.destroy();

Destroys the underlying peer connection and stops all BubbleHash periodic processes.

bubblehash.lookupmethodbubblehash.lookup(key, [value])

Looks up and optionally sets the value of a given `key`.

keystring

A string descriptor corresponding to a particular value.

[value]string object array number boolean

The value corresponding to `key` it can be any serializable Javascript data type.
